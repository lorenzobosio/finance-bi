RLS policy shape (repeat for EVERY table — RESEARCH Pattern 2):
  alter table public.<t> enable row level security;
  create policy "allowlist_all" on public.<t>
    for all to authenticated
    using      ( (select auth.jwt() ->> 'email') in ('<email1>','<email2>') )
    with check ( (select auth.jwt() ->> 'email') in ('<email1>','<email2>') );
  -- the (select ...) wrapper is the documented Supabase initplan-caching perf pattern.
  -- Hardcode the 2 emails in the policy for Phase 0 (RESEARCH A5); keep them in sync with
  -- the app-layer ALLOWED_EMAILS env (note this in a seed-migration comment).
