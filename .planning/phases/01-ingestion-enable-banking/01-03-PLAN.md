<objective>
Build the Enable Banking JWT signer, the zod-validated typed client, and the production form of `pnpm eb:connect` that persists the consent — turning the live SCA into durable, queryable state.

This is the Connect MVP slice: after this plan, a human runs `pnpm eb:connect` once (D-07: one-time browser SCA, no in-app admin page) and a single consent connects all the cash accounts (D-08); the database holds a real `connections` row (with the true expiry) plus one `accounts` row per exposed Revolut account (and a virtual investing-account row if the spike showed it is not PSD2-exposed). The EB app id + private key + session live in GitHub Secrets + `.env.local`, the key never committed (D-09). The cron in 01-04/01-05 then reads this session headlessly. This makes ING-05's "expires_at from the real API response" structurally true and gives the banner slice real data to read.

Purpose: SCA needs a human and a browser exactly once; isolating it here (and persisting session_id + expires_at to Postgres, the single source of truth) is the whole design. Validating every EB payload with zod at the boundary (V5) protects against malformed/hostile bank responses.
Output: jwt.ts, schemas.ts, client.ts, the persisting eb-connect.ts, and GREEN jwt.test.ts + connect.test.ts.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Enable Banking API → client.ts | Untrusted external JSON (sessions/transactions/balances) |
| eb-connect → Postgres (service_role) | Privileged write of consent + account rows |
| Operator terminal → logs | Risk of printing PII/secrets |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-06 | Tampering / DoS | EB response payloads | mitigate | zod-parse every response in client.ts at the boundary (V5); fail loudly on shape change |
| T-01-10 | Spoofing | RS256 JWT (machine auth) | mitigate | signEbJwt via jose with kid=appId; private key from env only, never logged (V2/V6) |
| T-01-11 | Information Disclosure | Logs (descriptions, IBANs, amounts, key) | mitigate | Log only counts/status; never full descriptions/amounts/IBANs/JWT/key (V7) |
| T-01-12 | Availability/Integrity | Silent consent expiry | mitigate | ConsentExpiredError on 401/403; connections.consent_status drives the loud banner (ING-05) — wired in 01-04/01-05 |
| T-01-SC | Tampering | jose install | mitigate | jose Approved in RESEARCH Package Legitimacy Audit; installed in 01-01 |
