# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Surface scope (narrow):** Phase 1 ships exactly **two global status banners** and nothing else. No pages, dashboards, charts, or admin UI (those are Phase 2+). The two banners are:

1. **Freshness banner (D-15 / ING-06)** — a global "data as of {date}" strip reflecting the last successful daily pull. States: `fresh`, `stale`, `unknown` (never-synced).
2. **Reconnect-needed banner (D-16 / ING-05)** — a passive (non-blocking) strip shown when bank consent has expired or a 403 occurred, prompting a re-run of `pnpm eb:connect`.

This contract defines only these two banners: placement, per-state severity/color, copy, icon, dismissibility, and date rendering. Everything else is out of scope for Phase 1.

## Placement & Layout

| Aspect | Decision |
|--------|----------|
| Region | A single global **layout slot** — a `<StatusBanners />` region rendered at the very top of the authenticated app shell, **above** page content, **full-bleed** (edge-to-edge), stacked vertically. Not the OS/browser chrome; not a floating toast. |
| Order when both visible | **Reconnect banner first (top)**, freshness banner second (below it). Reconnect is the higher-urgency, action-required state. |
| Scroll behavior | **Static** (scrolls with the page) for Phase 1 — not sticky. Keeps the implementation trivial and avoids covering content on the (yet-to-exist) dashboards. Stickiness can be revisited in Phase 2 when real pages exist. |
| Responsiveness | Full-width on all breakpoints. On mobile, text wraps; the reconnect action affordance drops below the message (stacks) rather than truncating. Mobile-first (Fernanda is the primary mobile user). |
| Always-mounted | The `<StatusBanners />` slot is always mounted; each banner self-decides visibility from its state (see per-banner rules). |

**Why a layout slot, not the header:** the current `layout.tsx` is bare (no app chrome exists yet in Phase 1). A dedicated top-of-shell slot satisfies ING-06's "every dashboard shows" requirement without coupling to a header that does not yet exist. This was explicitly delegated to Claude's discretion in CONTEXT (D-15 placement note).

## Banner State → Severity → Color Mapping

All colors derive from existing CSS variables (`globals.css`). **No new color tokens.** Light-mode only.

### Banner 1 — Freshness ("data as of {date}")

| State | When | Severity | Surface / border | Text | Icon (lucide) | Visible? |
|-------|------|----------|------------------|------|---------------|----------|
| `fresh` | Last successful pull within threshold (≤ **36h**) | info / neutral | `bg-muted` + `border-border` | `text-foreground` (date in `text-muted-foreground`) | `CircleCheck` | **Always shown** (trust signal — ING-06 "every dashboard shows") |
| `stale` | No successful pull in > **36h** (≈ 1 missed daily cron + grace) | warning | `bg-amber-50` + `border-amber-200` | `text-amber-900` | `TriangleAlert` | Shown (escalated) |
| `unknown` | Never synced — zero successful pulls on record | warning | `bg-amber-50` + `border-amber-200` | `text-amber-900` | `CircleHelp` | Shown |

**Amber rationale:** the neutral preset has no semantic "warning" token, only `--destructive` (red). Stale data is a caution, not an error, so it must NOT use destructive red. Use Tailwind's built-in `amber-50/200/900` utilities directly — this is the documented, allowed exception to the neutral-only palette (warning is a semantic need, not decoration). Amber is reserved exclusively for the `stale`/`unknown` freshness state.

**Threshold (36h):** the cron runs daily ~06:00 Europe/Berlin (D-11). 36h = one full daily cycle plus a half-day grace, so a single late/failed run flips the banner to `stale`. Threshold rendered in code as a constant, not hardcoded in copy.

### Banner 2 — Reconnect needed (consent expired / 403)

| State | When | Severity | Surface / border | Text | Icon (lucide) | Visible? |
|-------|------|----------|------------------|------|---------------|----------|
| `reconnect` | `connections.consent_status` expired OR last pull returned 401/403 | error (destructive) | `bg-destructive/10` + `border-destructive/30` | `text-destructive` (body in `text-foreground`) | `PlugZap` | Shown only when in this state |
| (hidden) | Consent valid / no 403 | — | — | — | — | Not rendered |

**Destructive rationale:** an expired consent means ingestion is broken and silently stops — the loudest passive state in the app. It maps to the existing `--destructive` token. It is still **passive / non-blocking** (D-16): it does not modal-block, does not disable the app, it informs and points to the fix.

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Gap between icon and text; gap between message and inline action |
| sm | 8px | Vertical padding inside each banner row |
| md | 16px | Horizontal padding inside each banner (left/right edge inset); gap when action stacks on mobile |
| lg | 24px | (reserved — not used by banners) |

Exceptions: none. Banners are compact: `py-2 px-4` (8px vertical / 16px horizontal), `gap-2` (8px) between icon and text. Stacked banners have **no** gap between them (each carries its own bottom border for separation).

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Banner message | 14px (`text-sm`) | 400 (regular) | 1.5 |
| Banner emphasis (state label, e.g. "Reconnect needed") | 14px (`text-sm`) | 600 (semibold) | 1.5 |
| Date token ("21 Jun 2026") | 14px (`text-sm`) | 500 (medium), `font-mono` (`--font-geist-mono`) | 1.5 |
| Action affordance ("Reconnect" link) | 14px (`text-sm`) | 500 (medium) | 1.5 |

Two weights for prose (400 + 600); 500 used only for the monospaced date and the action label so they read as interactive/data, not body copy. No heading/display type — banners have no headings.

**Date rendering:** absolute date in `font-mono`, format **`d MMM yyyy`** (e.g. `21 Jun 2026`) via `date-fns` (already in stack). For `fresh`/`stale`, optionally append a relative hint in parentheses using `muted-foreground` (e.g. `(2 days ago)`) — relative text is plain sans, the absolute date is the mono anchor. Never show a bare ISO timestamp.

## Color

Palette is the established neutral preset + the destructive token + a single reserved amber for warning. 60/30/10 applies to the app overall; for this two-banner surface the mapping is:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` (oklch(1 0 0), white) | Page behind the banners; fresh-banner text base |
| Secondary (30%) | `--muted` (oklch(0.97 0 0)) | `fresh` freshness banner surface |
| Accent (10%) | amber-50/200/900 (Tailwind) | **Only** the `stale` / `unknown` freshness state |
| Destructive | `--destructive` (oklch(0.577 0.245 27.325)) | **Only** the reconnect-needed banner (and at `/10` tint for its surface) |

Accent reserved for: the freshness banner's `stale` and `unknown` warning states **only**. Never used on the `fresh` state, never on the reconnect banner, never decoratively. Destructive reserved for: the reconnect-needed banner **only**.

## Copywriting Contract

All copy in English. Concise, factual, trust-building — no alarm, no jargon.

| Element | Copy |
|---------|------|
| Freshness `fresh` | `Data as of {date}.` — e.g. `Data as of 21 Jun 2026.` (optional muted suffix `(synced today)` / `(2 days ago)`) |
| Freshness `stale` | `Data may be out of date — last successful sync was {date}.` |
| Freshness `unknown` | `No data synced yet. The first import will appear here once the daily sync runs.` |
| Reconnect `reconnect` (emphasis label) | `Reconnect needed` |
| Reconnect `reconnect` (body) | `The bank connection has expired. Run \`pnpm eb:connect\` to restore the daily sync.` |
| Reconnect action affordance | label: `How to reconnect` (link/button — opens the short reconnect instructions; see Action below) |
| Primary CTA (phase-level) | `Reconnect` — the only action verb+noun in this phase; surfaces the `pnpm eb:connect` instruction. No destructive actions exist in Phase 1. |
| Empty state heading | `No data synced yet` (this is the freshness `unknown` state — the banner IS the empty state for Phase 1) |
| Empty state body | `The first import will appear here once the daily sync runs.` |
| Error state | `The bank connection has expired. Run \`pnpm eb:connect\` to restore the daily sync.` (the reconnect banner IS the error state) |
| Destructive confirmation | not applicable — Phase 1 has **no destructive actions** (no delete, no disconnect-from-UI; connection is managed via the local script per D-07) |

**Code/command rendering:** `pnpm eb:connect` renders as inline `<code>` (mono, `bg-muted`, subtle), so the user can copy it. It is an instruction, not a clickable button (there is no in-app admin page in the MVP — D-07).

## Interaction & Dismissibility

| Banner | Dismissible? | Behavior |
|--------|--------------|----------|
| Freshness `fresh` | No | Always visible; it is a passive trust indicator, not a notification. |
| Freshness `stale` / `unknown` | No | Stays until the underlying state resolves (a successful sync makes it `fresh`). Persistent on purpose — staleness must remain "loudly visible" (ING-06). |
| Reconnect `reconnect` | **Soft-dismiss per session** | A small `X` (lucide `X`, `text-muted-foreground`, 16px) collapses it for the current session (client state only — `sessionStorage`). It **reappears on next load** while the underlying expired/403 state persists. Never permanently dismissible — D-16 requires it stay visible until the consent is actually restored. |

**Reconnect action:** the `How to reconnect` affordance is a shadcn Button `variant="link"` size `sm` (or `variant="outline"` size `sm` if a stronger affordance is wanted) that reveals/links a short instruction (the `pnpm eb:connect` command + one line of context). No network call, no in-app reconnect flow in Phase 1 (consent is a local-script, browser SCA step — D-07). Keyboard-focusable; visible focus ring via the existing `--ring` token.

**Accessibility:**
- Freshness banner: `role="status"`, `aria-live="polite"` (non-urgent).
- Reconnect banner: `role="alert"`, `aria-live="assertive"` (action required).
- Icons are decorative (`aria-hidden`); meaning is carried by text, never color alone. Amber and destructive both pair an icon + a text label so the state is legible without color perception.
- Dismiss `X` has `aria-label="Dismiss reconnect notice"`.
