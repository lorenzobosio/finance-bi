## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser form → Server Action | The budget edit crosses from untrusted client into a 'use server' function; zod validates before any DB write |
| Server Action (WRITE plane) → budgets table | Writes through @supabase/ssr (anon + user JWT); the existing allowlist RLS on budgets authorizes — no service_role |
| RSC pages (READ plane) → mart views | supabase.from('v_*') under the user JWT; RLS/security_invoker enforces the allowlist |
| developer → public repo | pages + actions committed publicly — no € amounts, no PII, no service_role/DATABASE_URL |
