# Phase 2 — UI Design Contract

> Visual and interaction contract for the Core BI + house-as-business dashboards. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope:** the 5 dashboard pages (Home, Gastos/Spending, Cost Centers + Sublocação, Transações, Config) plus the shared app shell. Out of scope (do NOT design): €100k goal page / gamification / waterfall (Phase 3), PWA install (Phase 4), AI insight strip (Phase 5), ETF market value (Phase 6), notifications (Phase 7). Reference deferred surfaces only as disabled/placeholder nav items.

---

## North-Star Test (the contract every page is judged against)

The product must let the couple answer **4 questions in under a minute** (BI-05). Every design choice serves this:

1. **How far to €100k invested?** — read-only progress preview on Home (full goal page is Phase 3; here show cumulative `investimento` cost-basis as a KPI, no milestones/ETA).
2. **Did we hit €4k this month?** — €4k contribution status KPI.
3. **Did either person blow their budget?** — per-person budget-status KPI.
4. **What's the margin?** — margin % KPI (result ÷ revenue).

Audience split drives every responsive contract:
- **Fernanda — mobile-first, non-technical.** Home must be glanceable, single-column-stacking, large numbers, status carried by icon+label+color (never color alone).
- **Lorenzo — desktop, technical.** Data-dense multi-column grids, sortable tables, the Transações editor, drill-downs.

CV-quality bar: restrained Mercury/Ramp/Linear-calibre fintech aesthetic. A real visual system below — not vague guidance.

## Spacing Scale

8-point scale (Tailwind v4 default `--spacing` = 4px; use the multiples below only):

| Token | Value | Tailwind | Usage |
|-------|-------|----------|-------|
| xs | 4px | `1` | Icon-to-label gap, delta-chip inner padding |
| sm | 8px | `2` | Compact stacks, KPI label-to-value gap, table cell padding-y |
| md | 16px | `4` | Default element spacing, card inner padding (mobile) |
| lg | 24px | `6` | Card inner padding (desktop), gap between KPI cards, section padding |
| xl | 32px | `8` | Page padding (desktop), major layout gaps |
| 2xl | 48px | `12` | Section breaks between major blocks (P&L vs trend) |
| 3xl | 64px | `16` | Page-level top/bottom rhythm on tall dashboards |

**Exceptions:**
- Touch targets ≥ **44px** min height for all interactive controls on the mobile Home and the mobile bottom-nav (icon-only nav items get 44×44).
- Page horizontal padding is `16px` (mobile) → `32px` (≥`lg`). Content max-width `1280px` (`max-w-7xl`) centered for desktop dashboards; Home stays narrower (`max-w-md` mobile feel, expands to grid on desktop).
- Mobile bottom-nav bar height `56px` (fixed), with `env(safe-area-inset-bottom)` padding.

## Typography

`--font-sans` for everything except numerics; `--font-mono` for ALL money, percentages, dates, and period keys (`2026-06`). Exactly 2 weights: **400 (regular)** and **600 (semibold)**.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display (KPI value) | 32px (`text-3xl`) | 600 | 1.2 | The big number on a KPI card; mono |
| Heading (page/section) | 20px (`text-xl`) | 600 | 1.2 | Page title `h1`, card titles `h2` |
| Body | 16px (`text-base`) | 400 | 1.5 | Default copy, table values |
| Label / caption | 14px (`text-sm`) | 400 (semibold for KPI labels) | 1.4 | KPI label, secondary text (`text-muted-foreground`), table headers, helper text |

**Rules:**
- Money is ALWAYS `font-mono` with thousands separators and the `€` symbol (e.g. `€5,038`, `€100,000`). Use `Intl.NumberFormat('en-IE'|'de-DE', { style:'currency', currency:'EUR' })` — pick one and pin it; the comma-grouped `en-IE` form (`€5,038.00`) matches the existing `font-mono` money convention. No bare numbers for currency.
- Negative money renders with a leading minus and the loss color (below), never parentheses.
- Percentages mono, one decimal max (`12.4%`, `0%`).
- Dates: `date-fns` `format(d,'d MMM yyyy')` (`23 Jun 2026`) in mono — matches the existing FreshnessBanner. Period labels: `MMM yyyy` (`Jun 2026`).
- No more than these 4 sizes. No `text-2xl`/`text-4xl` except the single Home hero KPI may use `text-4xl` (36px) at 600 if visual hierarchy needs it — declared exception, Home only.

## Color

The base token set in `globals.css` is intentionally **greyscale** (neutral) — surfaces, cards, sidebar, and `chart-1..5` are all greys. For a finance-BI product the contract **adds a small, reserved semantic palette** for gain/loss/budget status. These are the ONLY chromatic colors in the app; everything structural stays greyscale (the Mercury/Ramp restraint).

| Role | Token / Value | Usage |
|------|---------------|-------|
| Dominant (60%) | `--background` `oklch(1 0 0)` (white) | Page background, primary surface |
| Secondary (30%) | `--card` / `--muted` / `--sidebar` (greys `oklch 0.97–0.985`) | Cards, sidebar/nav, table zebra, KPI card surfaces, chart neutral bars |
| Accent / primary (10%) | `--primary` `oklch(0.205 0 0)` (near-black) | Primary CTA fill, active nav item, focused control, progress-bar fill, key data emphasis |
| Destructive | `--destructive` `oklch(0.577 0.245 27.325)` (red) | Destructive actions only (delete rule, delete category) |

**Semantic finance palette (NEW — add as CSS vars in `globals.css` `:root` + `@theme inline`):**

| Semantic | Light value (suggested) | Reserved EXCLUSIVELY for |
|----------|------------------------|--------------------------|
| `--gain` | `oklch(0.62 0.17 150)` (green) | Positive margin, positive MoM delta, budget under cap, on-track €4k, positive sublet net, surplus result |
| `--loss` | `oklch(0.58 0.20 27)` (red, aligns to destructive hue) | Negative margin, negative MoM delta, **over-budget**, missed €4k, negative result/sublet net |
| `--warning` (amber) | `amber-50 / amber-200 / amber-900` Tailwind utilities (already in use) | Stale/unknown freshness, provisional-month flag, "approaching budget" (≥85%), reconnect banner — the documented amber exception, NOT a general accent |
| `--neutral-data` | `--muted-foreground` grey | "Not set" budgets, €0 empty months, Uncategorized slice, zero-delta |

**Hard color rules (carry over the Phase-1 contract):**
- **Color is never the only signal.** Every gain/loss/over-budget/status uses icon (lucide: `TrendingUp`/`TrendingDown`/`TriangleAlert`/`CircleCheck`) **and** text/label **and** color. Accessibility = WCAG AA contrast on all semantic text against its surface.
- Accent (near-black `--primary`) is reserved for: primary CTA, active nav item, progress-fill, focus ring — NOT for "all interactive elements" (secondary actions are `--secondary`/ghost).
- Charts: default series use the greyscale `--chart-1..5` tokens; gain/loss is the ONLY place chromatic green/red enters a chart (e.g. budget bar over cap turns `--loss`, under stays neutral/`--gain`). Do not rainbow-color categories — use a single neutral hue ramp + labels.
- The €4k / margin / budget / sublet "good vs bad" mapping: **good = `--gain`, bad = `--loss`, in-progress/partial = `--warning` or neutral.** Consistent across all 5 pages.

Accent reserved for: **primary CTA fill, active nav item, progress-bar fill, focus ring, single key emphasized value per card.**

## Page & Component Contracts

### 0. App Shell / Navigation (shared `(protected)/layout.tsx`)
- **Desktop (≥`lg`):** fixed left **sidebar** (~`240px`, `--sidebar` surface), brand at top, vertical nav list, active item = `--primary` text + subtle `--sidebar-accent` fill + left indicator. Content area to the right with `32px` padding, `max-w-7xl`.
- **Mobile (<`lg`):** **bottom tab bar** (fixed, 5 icon+label tabs, `56px` + safe-area), active tab = `--primary`. No hamburger (5 items fit a bottom bar; keeps Fernanda one-tap).
- **Freshness banner** mounts full-bleed at the very top of the shell content (above page title), on **every** page (`StatusBanners`). ReconnectBanner stacks above it when consent expired.
- **Nav items (order):** Home · Gastos · Cost Centers · Transações · Config. A disabled **Goal** placeholder may appear (greyed, "Phase 3") — optional.
- **Page header pattern:** `h1` (20px/600) + optional period switcher (month selector, MoM context) on the right.

### 1. Home — 4 headline KPI cards (BI-05, mobile-first)
- **Layout:** single column stack on mobile (cards full-width, `16px` gap), → 2×2 grid at `md`, → 4-across row at `xl`. Order maps to the 4 questions: **(1) €100k progress → (2) €4k this month → (3) per-person budget → (4) margin %.** Optional 5th/6th (savings-rate, months-of-reserve, D2-Discretion) render below the headline 4 in a secondary grid, visually lighter.
- **KPI card anatomy (single reusable `KpiCard`):**
  1. **Label** (14px/600, `text-muted-foreground`, uppercase optional) + small lucide glyph.
  2. **Big value** (Display 32px/600 mono) — the headline number (`€42,180`, `€4,000`, `18.2%`).
  3. **Status / delta row:** a delta chip — arrow icon + `+€320` or `▲ 4.1%` vs last month, colored `--gain`/`--loss`; OR a status pill ("On track" / "Missed" / "Over budget" / "Provisional") with icon.
  4. **Mini-viz (optional, per card):** €100k card → thin mini-progress bar (fill `--primary`, track `--muted`, % label). €4k card → progress-to-4000 bar. Budget card → two small stacked person bars (L / F) each green/amber/red. Margin card → 6-month mono sparkline (Recharts).
- **€100k card (Phase-2 limited):** show cumulative `investimento` cost-basis as the value + a progress bar toward €100,000 + "% to goal". NO milestones, NO ETA, NO streak (those are Phase 3). Label it plainly "Invested (cost basis)".
- **€4k card:** value = this month's `investimento` total; status pill: `≥€4,000` → "On track" `--gain` `CircleCheck`; `<€4,000` in a still-open month → "€{remaining} to go" `--warning`; closed month under → "Missed" `--loss`.
- **Budget card:** per-person worst-case status — if either Lorenzo or Fernanda is over budget → "Over budget" `--loss`; "not set" budgets are excluded from the status (never render a fake breach). Tapping → Cost Centers.
- **Margin card:** value = margin %; delta vs last month; `--gain` if positive & improving.

### 2. P&L / Margin (lives on Gastos or a dedicated panel; BI-01)
- Vertical waterfall-style **summary table or stacked bar**: Revenue (salary, `--gain` direction) → minus `investimento` (€4k legs, neutral, labeled "excluded from costs") → minus Costs → **= Result** (bold, `--gain`/`--loss`) → **Margin %**.
- **Sublocação enters as a single `Sublet net` line** (D2-07) — one row, its own icon, value `--gain`/`--loss`; the gross legs are NOT shown here (they live in the Sublocação view). A small "view details →" link jumps to the Sublocação P&L.
- `investimento` and `transferência` rendered visibly **excluded** (greyed, struck context or an "excluded" tag) so the user trusts the margin math.

### 3. Cost Centers + Sublocação (BI-02, D2-06/07/12)
- **Three budgeted-vs-actual cards/rows** (Lorenzo / Fernanda / Shared): horizontal bar, actual fill vs budget track. States:
  - Budget **not set** → grey bar labeled "Budget not set" + a "Set budget" link (NEVER a fake cap, D2-12).
  - Under budget → `--gain` (or neutral) fill, "{actual} of {budget}".
  - ≥85% → `--warning`. Over → `--loss` fill overflowing the track + "Over by €{x}" + `TriangleAlert`.
- **Category grain (D2-14):** each cost center expands to category sub-bars (Accommodation/Food/… ) if category-level budgets exist.
- **Sublocação profit-center view** (standalone, D2-08): its OWN mini-P&L card — Rent received (`--gain`) − Rent/utilities paid (`--loss`) = **Net** (bold). This is the only place sublet gross legs appear. Visually separated from the 3 cost centers (own section header "Sublocação — profit center").

### 4. Gastos / Spending (BI-03, D2-15)
- **Breakdown by category / account / person** — a segmented toggle (3 options) above a horizontal-bar breakdown (single neutral hue ramp + value labels; biggest first). **Uncategorized** always renders as its own slice/bar in `--neutral-data` grey with a "{n} to categorize →" link to Transações (D2-01 graceful degrade).
- **Category-as-%-of-revenue (D2-15):** a first-class view — table with sparkbars OR a single 100%-stacked bar of categories against revenue; each row "Food — €820 — 14% of revenue". Mono numbers. This answers "how much of our income each category eats".

### 5. Transações (CAT-04/05, D2-01/02/03)
- **Dense sortable table** (desktop-primary; mobile = stacked card rows). Columns: Date · Merchant/memo · Account · **Category** · **Cost center** · Amount (mono, `--loss` for outflow direction is fine but keep cost neutral; reserve red for over-budget — outflows can be neutral with a − sign).
- **Uncategorized rows** are surfaced first / flagged with an amber `TriangleAlert` dot + "Uncategorized" pill in `--neutral-data`.
- **Inline edit interaction:** clicking a row's Category or Cost center opens a popover/select:
  - Change category → updates **that one row immediately** (D2-03).
  - A checkbox/toggle "**Also create a rule** (apply to future {merchant})" → writes/updates a `rules` row that persists forward (D2-02). Copy must make "future-only" explicit.
  - Applying a rule to **past rows is a SEPARATE explicit button** — "Re-apply to {n} matching past transactions" — never automatic on save (D2-03/CAT-05). Show a confirm dialog with the count.
- Cost-center assignment uses the same persists-forward pattern (CAT-07 default + override).

### 6. Config (BI-06, D2-12/13/14)
- **Budgets editor:** list cost centers (and categories, D2-14) with an editable € input, starting **empty/€0** (D2-12). "Set from history" helper button (after ~1 month of data) prefills last month's actual or trailing average (D2-13). No hardcoded amounts.
- **Categories & rules management:** list/edit/delete categories (group = essential|desire|investment) and `rules` (priority, match, sets). Delete = destructive (red, confirm). Editing a rule offers the same "re-apply to past" explicit action.
- **Reconnect / consent** lives here too (existing Phase-1 banner surfaces; Config is where the user re-auths).

### 7. Comparability & first-class states (BI-04)
These are **first-class, designed states — not afterthoughts.** Every mart-backed view must handle:
- **Empty month → `€0`** rendered in `--neutral-data` grey (not blank, not "no data"). Bars collapse to an empty track.
- **Current partial month → "Provisional" amber pill** (`--warning`) next to the period label on every page; tooltip "Month in progress — figures will change."
- **YoY → "Insufficient history"** muted note until ~12 months of `dim_calendar`-joined data exist; show MoM instead.
- **Loading:** skeleton shimmer on cards/tables (shadcn skeleton), never layout shift.
- **Uncategorized:** always a visible, labeled grey slice — never crashes a breakdown (D2-01).
- **Stale / disconnected:** the existing amber FreshnessBanner + ReconnectBanner (ING-05/06).
