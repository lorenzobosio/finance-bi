# Phase 2 — UI Design Contract

> Visual and interaction contract for the Core BI + house-as-business dashboards. Generated by gsd-ui-researcher, verified by gsd-ui-checker, then revised against an external P1/P2/P3 review (2026-06-23).
>
> **Scope:** the 5 dashboard pages (Home, Gastos/Spending, Cost Centers + Sublocação, Transações, Config) plus the shared app shell. Out of scope (do NOT design): €100k goal page / gamification / waterfall-to-goal (Phase 3), PWA install (Phase 4), AI insight strip (Phase 5), ETF market value (Phase 6), notifications (Phase 7). Reference deferred surfaces only as disabled/placeholder nav items.

## North-Star Test (the contract every page is judged against)

The product must let the couple answer **4 questions in under a minute** (BI-05). Every design choice serves this:

1. **How far to €100k invested?** — read-only progress preview on Home (full goal page is Phase 3; here show cumulative `investimento` cost-basis as a KPI, no milestones/ETA).
2. **Did we hit €4k this month?** — €4k contribution status KPI.
3. **Did either person blow their budget?** — per-person budget-status KPI.
4. **What's the margin?** — margin % KPI = **(revenue − investimento − costs) ÷ revenue** (the €4k IS subtracted — "pay yourself first"; matches CLAUDE.md, confirmed 2026-06-23). Labeled **"% of net revenue"**.

Audience split drives every responsive contract:
- **Fernanda — mobile-first, non-technical.** Home must be glanceable, single-column-stacking, large numbers, status carried by icon+label+color (never color alone).
- **Lorenzo — desktop, technical.** Data-dense multi-column grids, sortable tables, the Transações editor, drill-downs.

CV-quality bar: restrained Mercury/Ramp/Linear-calibre fintech aesthetic. A real visual system below — not vague guidance.

## Charting (shadcn official + Tremor Raw, on Recharts 3.x)

**Recharts is pinned to `3.8.1` (EXACT, no `^`)** in `package.json` — the version Phase 0 already installed. This supersedes the planning-time "Recharts 2.x" note in CLAUDE.md (corrected to 3.x as part of this revision). shadcn's official chart component (`chart.tsx` / `ChartContainer`) is **Recharts-3-native** (shadcn PR #8486 "feat: update chart to recharts v3"), and current Tremor Raw blocks target Recharts 3 too — so the whole chart layer is on one version. No 2.x downgrade.

**Recharts-3 paste rules (apply when copying ANY shadcn/Tremor block):**
- Reference chart colors as `var(--chart-1)` — NOT `hsl(var(--chart-1))` (Recharts-3 change).
- `ChartContainer` MUST carry a height / `min-h-*` / `aspect-*` so Recharts' `ResponsiveContainer` measures on first render (else zero-height charts).
- Remove `layout` from `<Bar>` when the parent `<BarChart>` already sets it; use `ChartTooltip` `defaultIndex` for initial tooltip state.
- Every copied block is reviewed on paste (we own the source — no runtime registry).

**Component map (use these; do not hand-roll):**

| Surface | Component | Source | Notes |
|---|---|---|---|
| Gastos breakdown (category / account / person) | **BarList** | Tremor Raw | horizontal bars, value labels, biggest-first; single neutral hue ramp |
| Category-as-%-of-revenue (D2-15) | **CategoryBar** (100%-segmented) or BarList w/ % | Tremor Raw | each row "Food — €820 — 14 % of net revenue" |
| Budget vs actual (cost center + category) | **CategoryBar** / ProgressBar | Tremor Raw | track = budget, fill = actual; over-cap turns `--loss` |
| €100k progress, €4k progress | **ProgressBar** (ProgressCircle on €100k card) | Tremor Raw | fill `--primary`; % label |
| Margin 6-month KPI mini-trend | **SparkChart** or **pure-SVG sparkline** | Tremor Raw / bespoke | prefer pure-SVG (P3) to keep Recharts off the Home critical path |
| Balance / net-worth + months-of-reserve trend | **Area Chart** | shadcn (Recharts 3) | `ChartContainer` + `AreaChart`; neutral fill, mono axis |
| P&L waterfall | bespoke **stacked Bar** w/ transparent offset series | shadcn / Recharts 3 | no native waterfall block exists — see P&L section |
| €4k-per-day / freshness over month (optional) | **Tracker** | Tremor Raw | row of status cells |
| Category breakdown shape | **bars, NOT donut** (P3) | Tremor BarList | bars beat donuts for comparison + a11y; reserve DonutChart for nothing here |

**Chart chrome tokens:** gridlines = `--border`, **horizontal only** (no vertical grid), no chart border; axis/label text = `--muted-foreground` mono; series default to the greyscale `--chart-1..5` ramp — chromatic `--gain`/`--loss` enters a chart ONLY for good/bad emphasis (e.g. an over-cap bar). Never rainbow categories.

**Number formatting — single source of truth.** All money/percent go through central helpers in `src/lib/format.ts`:
- `formatEUR(n, decimals?)` → `'€' + new Intl.NumberFormat('de-DE', { minimumFractionDigits: decimals ?? 2, maximumFractionDigits: decimals ?? 2 }).format(n)` → **`€5.038,00`** (period thousands, comma decimal — native to DE & BR; `€` prefixed to match the app convention). Use **0 decimals on hero KPI values** (`€42.180`), **2 in tables/P&L**.
- `formatPct(n)` → de-DE one-decimal with the German space-before-`%` (`12,4 %`, `0 %`).
- NEVER call `Intl` ad-hoc. All numeric cells/axes use `tabular-nums` + `font-mono` so columns align (mono alone does NOT guarantee alignment).

| sm | 8px | `2` | Compact stacks, KPI label-to-value gap, table cell padding-y |

- Mobile bottom-nav bar height `56px` (fixed), with `env(safe-area-inset-bottom)` padding.

## Typography

`--font-sans` for everything except numerics; `--font-mono` + `tabular-nums` for ALL money, percentages, dates, and period keys (`2026-06`). Exactly 2 weights: **400 (regular)** and **600 (semibold)**.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display (KPI value) | 32px (`text-3xl`) | 600 | 1.2 | The big number on a KPI card; mono |
| Heading (page/section) | 20px (`text-xl`) | 600 | 1.2 | Page title `h1`, card titles `h2` |
| Body | 16px (`text-base`) | 400 | 1.5 | Default copy, table values |
| Label / caption | 14px (`text-sm`) | 400 (600 for KPI labels) | 1.4 | KPI label, secondary text (`text-muted-foreground`), table headers, helper text |

**Rules:**
- Money via central `formatEUR()` (see Charting): **de-DE separators, € prefixed → `€5.038,00`** (`€100.000`, `€820,50`). 0 decimals on hero KPI values; 2 in tables/P&L. `font-mono` + `tabular-nums`. No bare numbers, no ad-hoc `Intl`.
- Negative money renders with a leading minus and the loss color (below), never parentheses (`-€42,18`).
- Percentages via `formatPct()`, mono + `tabular-nums`, one decimal max, de-DE (`12,4 %`, `0 %`).
- Dates: `date-fns` `format(d,'d MMM yyyy')` (`23 Jun 2026`) in mono — matches the existing FreshnessBanner. Period labels: `MMM yyyy` (`Jun 2026`).
- **Mono discipline:** mono + `tabular-nums` is for ALIGNING numbers only (money, %, period keys, table numerics) — NEVER for prose, headings, or sentence copy.
- Exactly these 4 sizes — no more, no carve-outs. No `text-2xl`/`text-4xl` anywhere. The 32px Display (`text-3xl`) is the authoritative hero size; the €100k Home hero KPI gets emphasis through **structure** (col-span / elevated surface, see Home) + the 600 weight + whitespace — never a larger font.

## Color

The base token set in `globals.css` is intentionally **greyscale** (neutral). Surfaces are light: **`--card` is pure white `oklch(1 0 0)`**; `--muted`/`--secondary`/`--sidebar` are near-white greys (`0.97–0.985`). For a finance-BI product the contract **adds a small, reserved semantic palette** for gain/loss/budget status. These are the ONLY chromatic colors; everything structural stays greyscale (the Mercury/Ramp restraint).

| Role | Token / Value | Usage |
|------|---------------|-------|
| Dominant (60%) | `--background` `oklch(1 0 0)` (white) | Page background, primary surface |
| Secondary (30%) | `--card` (white) / `--muted` / `--sidebar` (greys) | Cards, sidebar/nav, table zebra, KPI surfaces, chart neutral bars |
| Accent / primary (10%) | `--primary` `oklch(0.205 0 0)` (near-black) | Primary CTA fill, active nav item, focused control, progress-bar fill, key data emphasis |
| Destructive | `--destructive` `oklch(0.577 0.245 27.325)` (red) | Destructive actions only (delete rule, delete category) |

**Semantic finance palette (NEW — add as CSS vars in `globals.css` `:root` + `@theme inline`).** Because cards are **white**, semantic colors used as **text** (deltas, "Net", "On track", "Over budget", "Uncategorized") MUST pass **WCAG AA (≥4.5:1) on white** — the original lighter green/red failed (~3.0–3.8:1). Two tiers per semantic: a **darker text/icon value** (AA on white) and a brighter **fill** value for bars/areas (where contrast-as-text doesn't apply):

| Semantic | Text value (AA on white) | Fill value (bars/areas) | Reserved EXCLUSIVELY for |
|----------|--------------------------|-------------------------|--------------------------|
| `--gain` | `oklch(0.52 0.15 150)` (green) | `oklch(0.62 0.17 150)` (or `/15` tint) | Positive margin, positive MoM delta, budget under cap, on-track €4k, positive sublet net, surplus result |
| `--loss` | `oklch(0.52 0.20 27)` (red) | `oklch(0.58 0.20 27)` (or `/15` tint) | Negative margin, negative MoM delta, **over-budget**, missed €4k (closed month only), negative result/sublet net |
| `--warning` | `oklch(0.47 0.12 75)` (amber text) | `amber-100/200` surfaces | Stale/unknown freshness, provisional-month flag, "approaching budget" (≥85%), reconnect/sync banner — the documented amber exception, NOT a general accent |
| `--neutral-data` | `oklch(0.50 0 0)` (grey, AA on white) | `--muted` | "Not set" budgets, €0 empty months, Uncategorized slice, zero-delta |

(All text-tier values target ≥4.5:1 on `oklch(1 0 0)`; verify with a contrast check at build and tune ±0.02 L if needed. `--neutral-data` text `oklch(0.50)` is intentionally darker than shadcn's default `--muted-foreground` `oklch(0.556)`, which is borderline on white.)

**Hard color rules:**
- **Color is never the only signal.** Every gain/loss/over-budget/status uses icon (lucide `TrendingUp`/`TrendingDown`/`TriangleAlert`/`CircleCheck`) **and** text/label **and** color.
- Accent (near-black `--primary`) is reserved for: primary CTA, active nav item, progress-fill, focus ring, single key emphasized value per card — NOT "all interactive elements" (secondary actions are `--secondary`/ghost).
- Charts: default series use greyscale `--chart-1..5`; gain/loss is the ONLY place chromatic green/red enters a chart. No rainbow categories.
- good = `--gain`, bad = `--loss`, in-progress/partial = `--warning` or neutral — consistent across all 5 pages.

## Page & Component Contracts

### 0. App Shell / Navigation (shared `(protected)/layout.tsx`)
- **Desktop (≥`lg`):** fixed left **sidebar** (~`240px`, `--sidebar` surface), brand at top, vertical nav, active item = `--primary` text + `--sidebar-accent` fill + left indicator. Content area right with `32px` padding, `max-w-7xl`.
- **Mobile (<`lg`):** **bottom tab bar** (fixed, 5 icon+label tabs, `56px` + safe-area), active tab = `--primary`. No hamburger (5 items fit; keeps Fernanda one-tap).
- **Freshness banner** mounts full-bleed at the very top of the shell content on **every** page (`StatusBanners`). ReconnectBanner stacks above it when consent expired.
- **Nav items (order):** Home · Gastos · Cost Centers · Transações · Config. A disabled **Goal** placeholder may appear (greyed, "Phase 3").
- **Month selector (MANDATORY, SHARED across all 5 pages):** one period switcher in the page header — current-month default, `‹ prev / next ›` arrows (next disabled at the current month), `MMM yyyy` label. Its value is **shared app state** (URL search param `?period=YYYYMM` + a context provider) so Provisional flagging, MoM deltas, and "did we hit €4k *this month*?" all key off the same selected period everywhere. Not optional — every mart-backed view reads it.
- **Page header pattern:** `h1` (20px/600) left + the shared month selector right.

### 1. Home — 4 headline KPI cards (BI-05, mobile-first)
- **Layout:** single column on mobile (cards full-width, `16px` gap) → 2×2 at `md` → 4-across at `xl`. Order maps to the 4 questions: **(1) €100k progress → (2) €4k this month → (3) per-person budget → (4) margin %.** Optional 5th/6th (savings-rate, months-of-reserve) render below the headline 4 in a lighter secondary grid.
- **€100k hero gets STRUCTURAL emphasis (NOT a bigger font — type stays 32px/600):** on desktop the €100k card spans 2 columns (or sits on a subtly **elevated surface** — `--card` + `ring`/`shadow-sm`) so it reads as hero; on mobile it's the first, full-width card. With only 2 weights, hierarchy comes from surface size + whitespace, never a 5th size.
- **Fernanda's mobile Home shows only the 4 household KPIs** — Sublocação is a separate profit center (D2-06) and does NOT appear as a Home KPI; it lives on Cost Centers. (Sublet net touches Home only indirectly, via the margin it feeds.)
- **KPI card anatomy (single reusable `KpiCard`):**
  1. **Label** (14px/600, `text-muted-foreground`) + small lucide glyph.
  2. **Big value** (Display 32px/600 mono) — the headline number.
  3. **Status / delta row:** a delta chip (arrow icon + `+€320` / `▲ 4,1 %` vs last month, `--gain`/`--loss`) OR a status pill ("On track" / "Missed" / "{name} over budget" / "Provisional") with icon.
  4. **Mini-viz (optional, per card):** €100k → ProgressCircle/bar toward 100k; €4k → ProgressBar to 4000; Budget → two small person bars (L/F); Margin → 6-month sparkline (pure-SVG preferred).
- **Drill-down (consistent):** the WHOLE card is tappable → its detail page (chevron on hover): €100k→Goal placeholder/Cost Centers · €4k→Cost Centers · Budget→Cost Centers · Margin→P&L/Gastos.
- **€100k card (Phase-2 limited):** cumulative `investimento` cost-basis as the value + progress bar toward €100.000 + "% to goal". NO milestones/ETA/streak (Phase 3). Label "Invested (cost basis)".
- **€4k card** — a red/"Missed" state can ONLY appear for a CLOSED past month; the current open month is NEVER red:
  - `≥ €4.000` → "On track" `--gain` `CircleCheck`.
  - `< €4.000` AND period = current (open) month → "**€{remaining} to go**" `--warning` (provisional).
  - `< €4.000` AND period = past (closed) month → "**Missed** — €{shortfall} short" `--loss`.
- **Budget card:** per-person status that NAMES who — "**{name} over budget**" (both → "Both over budget") `--loss`; all set budgets within cap → "On track" `--gain`. A DISTINCT state when no budgets are set → "**Budgets not set**" `--neutral-data` + "Set budgets →" (NOT "all ok" — never a false green or fake breach). "Not set" lines are excluded from the over/under calc. Tap → Cost Centers.
- **Margin card:** value = margin % ("% of net revenue"); delta vs last month; `--gain` if positive & improving. If the month includes a one-off bonus, annotate the delta (see Conventions).

### 2. P&L / Margin (lives on Gastos or a dedicated panel; BI-01)
- **Pinned visual: a WATERFALL** (bespoke stacked Bar on Recharts 3 — see Charting), NOT a plain table or generic stacked bar. Steps in order: **Revenue (net salary)** → **+ Sublet net** (one signed step, D2-07) → **− Investimento** (the €4k legs) → **− Costs** → **= Result**. Each step carries a mono data label; only the final **Result** bar is colored (`--gain`/`--loss`), intermediate steps stay neutral grey.
- **Formula (LOCKED, confirmed 2026-06-23):** `result = revenue − investimento − costs + sublet_net`; `margin = result ÷ revenue`, labeled **"% of net revenue"**. `revenue` = net salary (bonuses included but flagged one-off). `costs` EXCLUDE investimento, transferência, AND the sublet's own legs.
- **Sublocação enters as the single `Sublet net` step/line** (D2-07) — gross legs are NOT shown here (Sublocação view owns them). A "view details →" link jumps there.
- `investimento` and `transferência` get a visible **"excluded from costs"** tag. Note: investimento IS subtracted in the Result step, but it is NOT a "cost" — the tag communicates it's a separate, intentional line, so the margin math reads true.

### 3. Cost Centers + Sublocação (BI-02, D2-06/07/12)
- **Three budgeted-vs-actual rows** (Lorenzo / Fernanda / Shared): CategoryBar/ProgressBar, actual fill vs budget track. States:
  - Budget **not set** → grey bar "Budget not set" + "Set budget" link (NEVER a fake cap, D2-12).
  - Under budget → `--gain`/neutral fill, "{actual} of {budget}".
  - ≥85% → `--warning`. Over → `--loss` fill overflowing the track + "Over by €{x}" + `TriangleAlert`.
- **Category grain (D2-14):** each cost center expands to category sub-bars if category-level budgets exist.
- **Sublocação profit-center view** (standalone, D2-08): its OWN mini-P&L card — Rent received (`--gain`) − Rent/utilities paid (`--loss`) = **Net** (bold). The ONLY place sublet gross legs appear; visually separated under its own header "Sublocação — profit center". **Sublet costs NEVER enter the household "Costs" bucket** (D2-06/07).

### 4. Gastos / Spending (BI-03, D2-15)
- **Breakdown by category / account / person** — a segmented toggle (3 options) above a **BarList** (single neutral hue ramp + value labels; biggest-first; **bars, not a donut**). **Uncategorized** always renders as its own grey (`--neutral-data`) bar with "{n} to categorize →" (D2-01 graceful degrade).
- **Category-as-%-of-revenue (D2-15):** first-class — CategoryBar (100%-stacked) or BarList with %, each row "Food — €820 — 14 % of net revenue". Mono numbers. Answers "how much of our income each category eats".

### 5. Transações (CAT-04/05, D2-01/02/03)
- **Dense sortable table** (desktop-primary; mobile = stacked card rows). Columns: Date · Merchant/memo · Account · **Category** · **Cost center** · Amount (mono; outflows are neutral with a `−` sign — reserve red for over-budget, not every expense).
- **Uncategorized rows** surfaced first / flagged with an amber `TriangleAlert` dot + "Uncategorized" pill (`--neutral-data`).
- **Excluded rows:** `investimento` and `transferência` carry a muted **`excluded`** chip (excluded from costs/revenue) so the table reconciles visibly with the P&L.
- **Inline edit interaction:** clicking a row's Category or Cost center opens a popover/select:
  - Change → updates **that one row immediately** (D2-03).
  - Toggle "**Also create a rule for future {merchant}**" → writes/updates a `rules` row that persists forward (D2-02). Copy makes "future-only" explicit.
  - Applying a rule to **past rows is a SEPARATE explicit action** — "**Re-apply to {n} matching past transactions**" — never automatic on save (D2-03/CAT-05); confirm dialog shows the count.
- **Write path (LOCKED):** edits go through a **Next.js Server Action** (server-only). RLS = the 2-email **allowlist** (shared visibility), **NOT per-member** — both users may edit any row (cost center is an analytical label, not an access wall; CLAUDE.md/REQUIREMENTS). UI does an **optimistic update**, then reconciles. "Re-apply to past" is **idempotent** and returns a **server-side count** of affected rows; re-running changes nothing further.
- **Table mechanics:** **server-side keyset (cursor) pagination** ordered by `booking_date, id` (not offset); **fixed row height** (no layout shift as data loads); skeleton rows while loading. Virtualize only if a page exceeds ~200 rows.
- **Edit popover a11y:** full keyboard — `Enter`/`Space` opens, focus trapped inside, `Esc` closes and **restores focus** to the trigger cell; combobox follows the Radix listbox pattern.

### 6. Config (BI-06, D2-12/13/14)
- **Budgets editor:** list cost centers (and categories, D2-14) with an editable € input, starting **empty/€0** (D2-12). **"Set from history"** helper (after ~1 month of data) prefills last month's actual or trailing average (D2-13). No hardcoded amounts ever committed.
- **Categories & rules management:** list/edit/delete categories (group = essential|desire|investment) and `rules` (priority, match, sets). Delete = destructive (red, confirm). Editing a rule offers the same explicit "re-apply to past".
- **Reconnect / consent** lives here (existing Phase-1 banner; Config is where the user re-auths).

### 7. Comparability & first-class states (BI-04)
**First-class, designed states — not afterthoughts.** Every mart-backed view must handle:
- **First-use (global, forward-only):** until the first day of ingested data exists, the WHOLE app shows a single calm band — "**Synchronizing — your first data appears tomorrow**" — and KPI/breakdown surfaces render their `€0`/empty states beneath it, so an empty Home/Gastos reads as "new", never "broken". Clears automatically once any transaction lands.
- **Empty month → `€0`** in `--neutral-data` grey (not blank). Bars collapse to an empty track.
- **Current partial month → "Provisional" amber pill** (`--warning`) next to the period label on every page; tooltip "Month in progress — figures will change."
- **YoY → "Insufficient history"** muted note until ~12 months of `dim_calendar`-joined data exist; show MoM instead.
- **Loading:** skeleton shimmer on cards/tables (shadcn skeleton), never layout shift.
- **Uncategorized:** always a visible, labeled grey slice — never crashes a breakdown (D2-01).
- **Stale / disconnected:** the existing amber FreshnessBanner + ReconnectBanner (ING-05/06).

## Accessibility & i18n

- **Contrast:** all semantic text uses the AA text-tier on white (≥4.5:1); never color as the only signal (icon + label + color, always).
- **Charts have a textual alternative:** every progress/budget bar is a real `role="progressbar"` with `aria-valuenow/min/max` + a visible mono value; BarList/CategoryBar rows expose their value as text; the P&L waterfall and trend chart carry an associated data table or `aria-label` summary. A screen-reader user never depends on the SVG.
- **Keyboard:** sidebar + bottom-nav, the month-selector arrows, the Transações edit popovers, and all destructive dialogs are fully keyboard-operable with visible focus rings (`--ring`) and correct focus restore.
- **Language tags:** `<html lang="en">` (UI copy is English); Portuguese domain labels (`Sublocação`, `Gastos`) carry inline `lang="pt-BR"` so screen readers pronounce them correctly.
- **Motion:** honor `prefers-reduced-motion` — disable chart entrance animations, skeleton shimmer, and bar transitions when requested.

## Conventions & formulas (locked)

- **Two densities:** Home/KPIs use a calm density (lg card padding, 24px gaps); Transações + breakdown tables use a compact density (sm cell padding, 14px). Don't mix.
- **Months-of-reserve:** `liquid cash position ÷ trailing-3-month average monthly costs` (costs = the P&L `costs` bucket, excl. investimento/transferência/sublet). Render "{n} months".
- **MoM with a one-off bonus:** when a month's revenue includes a one-off bonus (`is_recurring=false`), annotate the MoM delta (asterisk/tooltip "includes one-off bonus") so the trend isn't misread as a new run-rate.
- **Sublet costs never enter the household "Costs" bucket** (D2-06/07) — only the Sublocação profit-center P&L holds them; the household P&L sees Sublocação as the single net line.
- **Mono discipline:** `font-mono` + `tabular-nums` for aligning numbers ONLY — never prose/headings.
